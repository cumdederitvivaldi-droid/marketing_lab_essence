=== 목적 ===
특정 광고 매체/Airbridge 원링크를 통해 유입된 신규 고객의 다음달 재이용 여부를
30일 기다리지 않고 D+7~14 선행 지표로 조기 판단하기 위한 대시보드.

=== CSV 컬럼 설명 ===
구분           : 행 유형 — 대시보드_변수 / 패널 / 개발자_전달사항
ID             : 고유 식별자 (VAR1~2 / P0~P5 / DEV1~5)
이름           : 패널명 또는 변수명
측정시점       : D+2, D+7, D+14, D+30 (해당 없을 경우 -)
목적_및_맥락   : 해당 패널/변수가 무엇을 측정하고 왜 중요한지
시각화_타입    : Grafana 패널 타입 (Time series / Bar chart / Table 등)
판단기준       : 수치 해석 기준 및 Threshold 설정 가이드
SQL_쿼리       : Grafana BigQuery 데이터소스에 입력할 SQL 쿼리 (BigQuery Standard SQL)
개발자_주의사항: 구현 시 반드시 확인해야 할 사항

=== 대시보드 구성 요약 ===
Grafana 변수 (Variables)
  - VAR1: $referral_channels  — 실험군 채널 Multi-select (쿼리로 목록 자동 조회)
  - VAR2: $cohort_start_date  — 코호트 집계 시작일 (Text input, 기본값 2026-01-01)

패널 6개
  - P0: 주별 코호트 모수          — Bar chart, 모수 200명 미만 시 경고
  - P1: [D+2] 수거 완료율         — Time series, 전체 평균 대비 -5%p 이하 경고
  - P2: [D+7] 7일 재주문율 ⭐     — Time series + Threshold (50%↑ 양호 / 30%↓ 위험)
  - P3: [D+14] 2회 이상 이용율    — Time series
  - P4: [D+14] 구독 전환율        — Time series, 3% 이상 양호
  - P5: [D+30] 첫 달 종합 지표    — Table 패널 (avg_orders / 결제율 / 리뷰율 / 구독율)

개발자 전달사항 5개
  - DEV1: receipt.status 결제완료 값 확인 및 P5 쿼리에 조건 추가 (TODO)
  - DEV2: Grafana 변수 포맷 — ${referral_channels:singlequote} :singlequote 필수
  - DEV3: 코호트 성숙도 필터 — 각 측정 기간이 경과한 코호트만 포함 처리됨
  - DEV4: Grafana 시계열 인식 — TIMESTAMP(cohort_week) AS time 으로 처리됨
  - DEV5: 실험군+전체 평균 동시 표시 — CASE WHEN 그루핑으로 처리됨

=== 선행 지표와 후행 지표 관계 ===
D+2  수거 완료율      → 광고-서비스 핏 (타겟 품질 측정)
D+7  7일 내 재주문율  → 다음달 재이용 가장 강력한 예측 지표 (단일 지표로 조기 판단 가능)
D+14 2회 이상 이용율 → 서비스 습관화 여부
D+14 구독 전환율      → 장기 리텐션 강력한 예측 인자 (재이용 그룹 6배 차이)
D+30 종합 지표        → 코호트 품질 최종 평가
