# 서버 관리

> 대상: `dev@covering.app` 멤버 (sudo 필요)
>
> **🔴 필수 규칙:**
> - `sudoers` 파일 직접 수정 금지 — 반드시 `/etc/sudoers.d/` 하위에 파일 추가
> - `/etc/profile.d/covering-labs-setup.sh` 수정 시 반드시 테스트 후 적용 (로그인 스크립트 오류 시 전체 사용자 접속 장애)
> - 시스템 패키지 설치(`apt install`)는 dev 팀 내 합의 후 진행

## 사용자 환경 관리

### 자동 세팅

첫 SSH 접속 시 자동 실행 (스크립트: `/etc/profile.d/covering-labs-setup.sh`):
1. 홈 디렉토리 바로가기 생성 (`/home/이름` → `/home/이름_covering_app`)
2. Cloud Storage 폴더 생성 (`gs://covering-labs/이름/`)

> 마커 파일 `~/.covering_labs_init`이 있으면 실행 안 함. 삭제하면 다음 접속 시 재실행.

### 자동 생성이 안 된 경우

```bash
# 바로가기 수동 생성
sudo ln -s /home/새사용자_covering_app /home/새사용자

# GCS 폴더 수동 생성
echo "" | gsutil cp - gs://covering-labs/새사용자/.keep
```

### covering-dev 그룹 관리

```bash
# 멤버 추가 (/shared 쓰기 권한)
sudo usermod -aG covering-dev 사용자명

# 현재 멤버 확인
getent group covering-dev
```

## Public VM (covering-labs-public) 관리

> `covering-labs-public` VM은 VPN 없이 공개 접근 가능한 서버입니다. 기본 구조는 private VM과 동일하지만, Site-to-Site VPN이 연결되지 않아 내부 AWS 리소스 접근은 불가합니다.

| 항목 | 값 |
|---|---|
| 인스턴스명 | `covering-labs-public` |
| 외부 IP | `34.64.144.174` |
| Zone | `asia-northeast3-a` |
| 공개 도메인 | `public-labs.covering.app` |
| 인스턴스 SA | `covering-labs-public@covering-app-ccd23.iam.gserviceaccount.com` |
| Deploy SA | `covering-labs-public-deploy@covering-app-ccd23.iam.gserviceaccount.com` |
| 방화벽 | HTTP/HTTPS 전체 공개 (0.0.0.0/0), VPN 불필요 |

### SSH 접속

```bash
# gcloud CLI 사용
gcloud compute ssh covering-labs-public --zone asia-northeast3-a --project covering-app-ccd23
```

> 대상: GitHub Actions로 배포된 Next.js / NestJS 앱들은 SA 유저(`sa_109369409955768144646`)의 PM2로 실행됩니다.

```bash
# 실행 중인 앱 목록
sudo -u sa_109369409955768144646 pm2 list

# 특정 앱 로그
sudo -u sa_109369409955768144646 pm2 logs 앱이름

# 앱 재시작
sudo -u sa_109369409955768144646 pm2 restart 앱이름

# 앱 중지

```

## batch 앱 스케줄 관리

batch 타입 앱은 SA 유저(`sa_109369409955768144646`)의 crontab에 자동 등록됩니다.

```bash
# 등록된 batch 스케줄 확인
sudo cat /var/spool/cron/crontabs/sa_109369409955768144646

# 특정 앱 로그 확인
tail -f /shared/apps/[앱이름]/logs/batch.log
```

> batch 앱을 수동으로 crontab에서 삭제하지 마세요. 앱 폴더를 git rm 후 push하면 자동으로 제거됩니다.

## nginx 앱 라우팅

nginx는 `/shared/nginx-confs/` 안의 `.conf` 파일을 자동으로 읽어 앱별 경로를 라우팅합니다.

```
https://labs.covering.app/앱이름  →  /shared/nginx-confs/앱이름.conf  →  localhost:포트
```

> 포트 80/443은 **GCP 방화벽(L4) + nginx(L7) 이중으로 VPN 전용**입니다. AWS Client VPN(`covering-vpn-v2`) 연결 시에만 접근 가능. 자세한 내용은 `docs/04_권한과_보안.md` 방화벽 섹션 참조.

```bash
# nginx 상태 확인
sudo systemctl status nginx --no-pager | head -5

# 설정 문법 검사
sudo nginx -t

# 설정 리로드 (covering-dev 그룹은 sudo 없이 가능)
sudo nginx -s reload

# 앱별 nginx 설정 확인
cat /shared/nginx-confs/앱이름.conf

# 포트 배정 현황
cat /shared/port-registry.json
```

## 재부팅 가이드

### 재부팅해도 유지되는 것 (추가 작업 불필요)

| 항목 | 저장 위치 | 비고 |
|---|---|---|
| 사용자 crontab | `/var/spool/cron/` | 자동 재시작 |
| 홈 디렉토리 + 바로가기 | `/home/` | 그대로 유지 |
| `/shared` + 권한 | 디스크 | 그대로 유지 |
| `covering-dev` 그룹 | `/etc/group` | 그대로 유지 |
| Ops Agent | systemd enabled | 자동 시작 |
| cron 서비스 | systemd enabled | 자동 시작 |
| nginx | systemd enabled | 자동 시작 |
| PM2 앱들 | `pm2-sa_109369409955768144646.service` | systemd로 자동 복구 |
| 자동 세팅 스크립트 | `/etc/profile.d/` | 그대로 유지 |
| sudoers 설정 | `/etc/sudoers.d/` | 그대로 유지 |
| OS Login / 스코프 / IAM | GCP 레벨 | 디스크와 무관 |

### 재부팅 시 사라지는 것

| 항목 | 복구 방법 |
|---|---|
| tmux 세션 | `tmux new -s 이름`으로 재시작 |
| `/tmp` 파일 | 다시 생성 |
| 외부 IP | 변경될 수 있음 → `gcloud compute instances list` |

### 재부팅 후 확인

```bash
sudo systemctl status google-cloud-ops-agent --no-pager | head -5
sudo systemctl status cron --no-pager | head -5

for user in $(cat /etc/passwd | grep '/home' | cut -d: -f1); do
  cron=$(sudo crontab -u $user -l 2>/dev/null)
  if [ -n "$cron" ] && ! echo "$cron" | grep -q 'no crontab'; then
    echo "[$user] crontab OK"
  fi
done

df -h /
```

**요약: 재부팅 후 추가 세팅 불필요.** tmux만 재시작.

## 설치된 스크립트 / 설정 파일

| 파일 | 용도 | 수정 가능 |
|---|---|---|
| `/etc/motd` | SSH 로그인 시 표시되는 안내 메시지 (AI 문서 참조 가이드 포함) | `dev@` |
| `/etc/profile.d/covering-labs-setup.sh` | SSH 첫 접속 시 자동 세팅 | `dev@` |
| `/etc/google-cloud-ops-agent/config.yaml` | Cloud Logging 수집 설정 | `dev@` |
| `/etc/sudoers.d/covering-labs-symlink` | 일반 유저 바로가기 생성 허용 | `jun@` (위험) |
| `/etc/sudoers.d/covering-labs-nginx` | `covering-dev`에 nginx reload sudo 허용 | `jun@` (위험) |
| `/etc/nginx/sites-available/covering-labs` | nginx 기본 서버 블록 (`/shared/nginx-confs/` 포함) | `dev@` |
| `/etc/nginx/sites-available/labs.covering.app` | `labs.covering.app` HTTPS 서버 블록 (certbot 관리) | certbot 자동 |
| `/shared/apps/` | 배포된 앱 코드 (GitHub Actions가 자동 관리) | GitHub Actions |
| `/shared/nginx-confs/` | 앱별 nginx 위치 설정 (배포 시 자동 생성) | GitHub Actions |
| `/shared/scripts/deploy-app.sh` | 배포 스크립트 (빌드 → PM2 → nginx → Slack) | GitHub Actions |
| `/shared/scripts/undeploy-app.sh` | 제거 스크립트 (PM2 삭제 → nginx conf 제거 → 포트 반환 → Slack) | GitHub Actions |
| `/shared/port-registry.json` | 앱↔포트 매핑 테이블 (자동 관리) | GitHub Actions |
| `/shared/.env` | 배포 환경변수 (`SLACK_BOT_TOKEN` 등) | `jun@` 수동 |
| `~/.covering_labs_init` | 자동 세팅 완료 마커 | 본인 |
| 사용자 `crontab` | 개인 스케줄러 | 본인 (dev는 sudo로 확인) |

### `/etc/profile.d/covering-labs-setup.sh`

```bash
#!/bin/bash
FULL_USER=$(whoami)
SHORT_USER=$(echo $FULL_USER | sed 's/_covering_app$//')
MARKER="$HOME/.covering_labs_init"

if [ ! -f "$MARKER" ]; then
  if [ "$FULL_USER" != "$SHORT_USER" ] && [ ! -e "/home/$SHORT_USER" ]; then
    sudo ln -s "/home/$FULL_USER" "/home/$SHORT_USER" 2>/dev/null
  fi

  gsutil ls "gs://covering-labs/${SHORT_USER}/" > /dev/null 2>&1
  if [ $? -ne 0 ]; then
    echo "" | gsutil cp - "gs://covering-labs/${SHORT_USER}/.keep" 2>/dev/null
  fi

  touch "$MARKER"
fi
```

### `/etc/google-cloud-ops-agent/config.yaml`

```yaml
logging:
  receivers:
    syslog:
      type: files
      include_paths:
        - /var/log/syslog
    auth:
      type: files
      include_paths:
        - /var/log/auth.log
  service:
    pipelines:
      default_pipeline:
        receivers: [syslog, auth]

metrics:
  receivers:
    hostmetrics:
      type: hostmetrics
  service:
    pipelines:
      default_pipeline:
        receivers: [hostmetrics]
```
