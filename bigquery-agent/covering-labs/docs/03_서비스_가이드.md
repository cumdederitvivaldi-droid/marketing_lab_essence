# 서비스 가이드

> 대상: 모든 `@covering.app` 구성원
> 서버에서 사용 가능한 GCP 서비스 — 키 파일 없이 바로 사용

---

## 사용 가능한 서비스

| 서비스 | 용도 | 추가 설정 |
|---|---|---|
| Google Sheets | 스프레드시트 읽기/쓰기 | 시트마다 SA에 공유 필요 |
| BigQuery | 데이터 조회 (읽기 전용) | 데이터셋마다 SA에 READER 등록 필요 (관리자) |
| Cloud Storage | 파일 저장/공유 | 없음 (바로 사용) |
| GitHub | 코드 clone/pull | 개인 토큰 설정 (→ [02_이용_가이드.md](02_이용_가이드.md)) |

> 모든 GCP 서비스는 키 파일 없이 `google.auth.default()`로 인증됨. 인스턴스에 스코프가 설정되어 있기 때문.

## AWS 내부 서비스 접근

GCP ↔ AWS 간 Site-to-Site VPN이 연결되어 있어서, AWS Private Subnet에 있는 내부 서비스에도 접근 가능합니다.

| 서비스 | URL | 용도 |
|---|---|---|
| Grafana | `https://grafana.covering.app` | BI 대시보드 (BigQuery 시각화) |
| Admin API | `https://admin-api.covering.app` | 백오피스 API |
| Admin Web | `https://admin.covering.app` | 백오피스 웹 |

```bash
# 서버에서 확인
curl https://grafana.covering.app
curl https://admin-api.covering.app
curl https://admin.covering.app
```

> 이 서비스들은 AWS Internal ALB 뒤에 있으며, DNS가 Private IP(`10.0.x.x`)를 반환합니다. GCP 인스턴스에서 VPN 터널을 경유하여 접근합니다. 상세: [covering-docs/infrastructure/data-pipeline/02_네트워크.md](https://github.com/covering-app/covering-docs/blob/main/infrastructure/data-pipeline/02_네트워크.md)

## Google Sheets

### 사전 설정 (시트마다 한 번)

1. Google Sheets에서 시트 열기 → 우측 상단 **"공유"**
2. `covering-labs@covering-app-ccd23.iam.gserviceaccount.com` 입력
3. "편집자" 또는 "뷰어" 선택 → 공유

> **🔴 공유하지 않은 시트는 접근 불가.** Sheets는 Google Drive 기반이라 시트 하나하나 서비스 계정에 공유해야 합니다. 공유 대상: `covering-labs@covering-app-ccd23.iam.gserviceaccount.com`

### Python 코드

```python
import gspread
import google.auth

creds, _ = google.auth.default(scopes=[
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/drive",
])
gc = gspread.authorize(creds)

# 읽기
sheet = gc.open("시트 이름").sheet1
data = sheet.get_all_records()
print(data)

# 쓰기
sheet.update_cell(1, 1, "Hello")
sheet.append_row(["A", "B", "C"])
```

## BigQuery

### 접근 가능 데이터셋 (읽기 전용)

| 데이터셋 | 접근 | 비고 |
|---|---|---|
| `secure_dataset` | ✅ | |
| `ads_data` | ✅ | |
| `airbridge_dataset` | ✅ | |
| `bag_delivery` | ✅ | |
| `cx_data` | ✅ | |
| `mixpanel` | ✅ | |
| `product` | ✅ | |
| `spot` | ✅ | |
| `secure_dataset_gcp_sa_discoveryengine` | ✅ | |
| `public` | ❌ | 의도적 제외 |

> **🔴 SELECT(조회)만 가능.** INSERT, DELETE, CREATE TABLE 등 쓰기 작업은 차단됨.
> 새 데이터셋 접근이 필요하면 관리자(jun)에게 요청.

### Sheets vs BigQuery 차이

| | Google Sheets | BigQuery |
|---|---|---|
| 접근 권한 관리 | 시트마다 SA에 공유 | 데이터셋마다 SA를 READER로 등록 |
| 누가 설정 | 시트 소유자 (본인 가능) | 관리자 |
| 접근 범위 | 공유한 시트만 | READER 등록된 데이터셋만 |
| 읽기/쓰기 | 둘 다 가능 | 읽기만 가능 |

### Python 코드

```python
from google.cloud import bigquery

client = bigquery.Client()

# 테이블 목록 확인
tables = list(client.list_tables("covering-app-ccd23.secure_dataset"))
for t in tables:
    print(t.table_id)

# 데이터 조회 (필요한 컬럼만 선택 + LIMIT 필수 — SELECT * 금지)
query = """
    SELECT id, created_at
    FROM `covering-app-ccd23.secure_dataset.auth`
    LIMIT 10
"""
df = client.query(query).to_dataframe()
print(df)
```

## Cloud Storage (`gs://covering-labs`)

파일 저장/공유를 위한 공용 버킷. 추가 설정 없이 바로 사용 가능.

**용도:**
- 배치 결과물(CSV, JSON 등) 저장
- 팀원 간 대용량 파일 공유
- 스크립트에서 생성한 데이터 보관

### 사용자별 폴더

SSH 첫 접속 시 자동 생성됨:

```
gs://covering-labs/
├── jun/       ← jun@covering.app
├── beige/     ← beige@covering.app
├── joy/       ← joy@covering.app
└── (새 사용자 접속 시 자동 생성)
```

> **🔴 자기 이름 폴더만 사용하세요.** Cloud Storage는 폴더별 권한 분리가 불가능합니다 (버킷 단위만 가능). 기술적으로 다른 사람 폴더에도 쓸 수 있지만, **절대 다른 사람 폴더에 파일을 올리거나 삭제하지 마세요.**

### GCP 콘솔에서 사용

브라우저에서 파일 조회/업로드/다운로드 가능:
https://console.cloud.google.com/storage/browser/covering-labs?project=covering-app-ccd23

### 서버에서 사용 (gsutil)

```bash
# 내 폴더에 업로드
gsutil cp ./결과.csv gs://covering-labs/내이름/결과.csv

# 다운로드
gsutil cp gs://covering-labs/내이름/결과.csv ./결과.csv

# 내 폴더 목록 확인
gsutil ls gs://covering-labs/내이름/

# 폴더 통째로 업로드
gsutil -m cp -r ./output/ gs://covering-labs/내이름/output/

# 삭제
gsutil rm gs://covering-labs/내이름/파일.txt
```

### Python에서 사용

```python
from google.cloud import storage

client = storage.Client()
bucket = client.bucket("covering-labs")

# 업로드
blob = bucket.blob("내이름/data.csv")
blob.upload_from_filename("./data.csv")

# 다운로드
blob = bucket.blob("내이름/data.csv")
blob.download_to_filename("./data.csv")

# 목록
for blob in client.list_blobs("covering-labs"):
    print(blob.name)
```
