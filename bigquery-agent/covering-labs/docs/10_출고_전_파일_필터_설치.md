# 출고 전 파일 필터 설치 가이드

> 대상: 비개발자, 운영 담당자, Claude Code / Codex 사용자
> 목적: `release-file-guard`를 개인 로컬에 설치해서 commit / push 전에 불필요한 파일을 자동으로 한 번 더 걸러내기

---

## 이 문서로 해결되는 것

이 가이드는 아래 상황을 위한 문서입니다.

- "SKILL.md 하나만 주면 설치되는 거야?"에 대한 실제 설치 절차가 필요할 때
- 비개발자가 개인 Mac/로컬 환경의 Claude Code 또는 Codex에 출고 전 파일 필터를 붙이고 싶을 때
- `배포해줘`, `git push`, `gh pr create` 직전에 정책 문서/임시 로그/민감 파일이 섞여 들어가는 실수를 줄이고 싶을 때

핵심은 이겁니다.

> `SKILL.md` 하나만으로는 부족합니다.
> skill 본문 + hook helper + 설정 연결이 같이 있어야 자동 트리거가 됩니다.

---

## 설치 방식 2가지

### 방법 A — 가장 쉬운 방법 (권장)

covering-labs repo가 내 컴퓨터에 있다면 아래 한 줄만 실행합니다.

```bash
cd /path/to/covering-labs
bash .claude/skills/release-file-guard/install-global.sh
```

이 명령은 아래를 자동으로 설치합니다.

- `~/.claude/skills/release-file-guard/SKILL.md`
- `~/.claude/hooks/release-file-guard.py`
- `~/.codex/skills/release-file-guard/SKILL.md`
- `~/.codex/hooks/release-file-guard.py`
- `~/.claude/settings.json` hook 연결
- `~/.codex/hooks.json` hook 연결

### 방법 B — 수동 설치

자동 설치를 쓰기 어렵다면 아래 파일을 직접 복사합니다.

```text
복사 원본
- .claude/skills/release-file-guard/SKILL.md
- .claude/skills/release-file-guard/INSTALL.md
- .hooks/release-file-guard.py
- .codex/skills/release-file-guard/SKILL.md
- .codex/skills/release-file-guard/INSTALL.md
```

그리고 아래 경로에 둡니다.

```text
Claude
- ~/.claude/skills/release-file-guard/SKILL.md
- ~/.claude/skills/release-file-guard/INSTALL.md
- ~/.claude/hooks/release-file-guard.py

Codex
- ~/.codex/skills/release-file-guard/SKILL.md
- ~/.codex/skills/release-file-guard/INSTALL.md
- ~/.codex/hooks/release-file-guard.py
```

그 다음 `settings.json` / `hooks.json`에 hook command를 연결합니다.

---

## Claude 설정 예시

`~/.claude/settings.json` 안에 아래 command hook이 있어야 합니다.

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python3 \"$HOME/.claude/hooks/release-file-guard.py\""
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "python3 \"$HOME/.claude/hooks/release-file-guard.py\""
          }
        ]
      }
    ]
  }
}
```

## Codex 설정 예시

`~/.codex/hooks.json` 안에 아래 command hook이 있어야 합니다.

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python3 \"$HOME/.codex/hooks/release-file-guard.py\"",
            "statusMessage": "출고 전 파일 필터 확인 중"
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "python3 \"$HOME/.codex/hooks/release-file-guard.py\"",
            "statusMessage": "출고 전 파일 필터 확인 중"
          }
        ]
      }
    ]
  }
}
```

**중요:** Codex는 대상 repo에서 hooks 기능이 켜져 있어야 합니다.
covering-labs는 이미 `.codex/config.toml`에 아래 설정이 있습니다.

```toml
[features]
codex_hooks = true
```

다른 repo에서 쓰려면 그 repo에도 같은 설정을 넣어야 합니다.

## 설치 후 검증

### 1차 검증 — prompt trigger

Claude Code 또는 Codex에 이렇게 말해봅니다.

```text
배포해줘
```

정상이라면 `[RELEASE-FILE-GUARD]` 로 시작하는 경고가 먼저 떠야 합니다.

### 2차 검증 — shell trigger

repo 안에서 아래 명령을 실행하기 직전에도 분류 경고가 떠야 합니다.

```bash
git commit -m "test"
git push origin <branch>
gh pr create
```

### 3차 검증 — 분류 결과

경고 표에서 아래처럼 보여야 정상입니다.

```text
ALLOW   앱 코드/필수 설정
REVIEW  앱 내부 정책 md, 루트 임의 markdown, .claude/.codex 변경
REJECT  .env, 로그, 덤프, 압축파일, build/cache 산출물
```

## 비개발자에게 전달할 때

비개발자에게는 아래 중 하나로 전달하세요.

### 전달 방식 1 — 가장 권장

- 이 문서 (`docs/10_출고_전_파일_필터_설치.md`)
- repo 경로
- 실행 명령 한 줄

```bash
bash .claude/skills/release-file-guard/install-global.sh
```

### 전달 방식 2 — 폴더 단위 전달

다음 폴더 전체를 같이 전달합니다.

```text
.claude/skills/release-file-guard/
.codex/skills/release-file-guard/
.hooks/release-file-guard.py
```

**주의:** `SKILL.md` 하나만 보내면 자동 트리거는 동작하지 않습니다.

---

## 업데이트 / 제거

### 업데이트

repo 최신본을 받은 뒤 다시 실행합니다.

```bash
bash .claude/skills/release-file-guard/install-global.sh
```

### 제거

```bash
rm -rf ~/.claude/skills/release-file-guard
rm -f ~/.claude/hooks/release-file-guard.py
rm -rf ~/.codex/skills/release-file-guard
rm -f ~/.codex/hooks/release-file-guard.py
```

그리고 `~/.claude/settings.json`, `~/.codex/hooks.json`에서
`release-file-guard.py` command hook을 삭제합니다.
