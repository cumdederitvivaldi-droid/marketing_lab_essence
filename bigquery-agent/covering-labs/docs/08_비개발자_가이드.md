## 💡 AI에게 요청하는 법 (Prompting)

AI는 구체적일수록 더 정확한 결과를 냅니다. 아래의 "좋은 예"를 참고하여 요청하세요.

| 요청 유형 | 나쁜 예 | 좋은 예 |
|---|---|---|
| **배치 (자동화)** | "매일 매출 요약해서 보내줘" | "매일 아침 9시에 [구글 시트 URL]의 '판매' 시트 데이터를 읽어서, 어제 총 매출액을 계산한 뒤 [슬랙 웹훅 URL]로 메시지를 보내주는 배치 프로그램을 만들어줘." |
| **웹 (대시보드)** | "대시보드 하나 만들어줘" | "우리 팀의 프로젝트 진행 상황을 보여주는 Next.js 대시보드를 만들어줘. 상단에는 전체 진행률 그래프를 보여주고, 하단에는 각 팀원별 할 일 목록을 표 형태로 보여주면 좋겠어." |
| **수정 요청** | "안 돼, 고쳐줘" | "배포 후에 확인해 보니 슬랙 메시지에 매출액이 원화(₩)로 표시되지 않아. 금액 뒤에 '원'을 붙이고 콤마(,) 처리를 해서 다시 배포해줘." |

## 🛠️ 필수 준비물

AI가 작업을 시작하기 전에 미리 준비해 두면 좋은 정보들입니다.

1.  **Google Sheets URL**: 데이터를 읽거나 써야 하는 시트의 주소
2.  **Slack Webhook URL**: 알림을 보낼 채널의 웹훅 주소 (Slack 앱 → 해당 채널 → Integrations → Incoming Webhooks → Add)
3.  **BigQuery 테이블 이름**: 데이터베이스를 사용해야 하는 경우
4.  **실행 주기**: 배치 작업의 경우 (예: "매주 월요일 오전 10시")

## 🔒 민감한 정보(Secret) 관리법

API 키나 슬랙 주소 같은 중요한 정보는 **절대 채팅창에 입력하거나 코드에 직접 넣지 마세요.**

환경변수 파일(`/shared/.env`)은 보안상 관리자(`jun@covering.app`)만 수정할 수 있습니다. AI가 코딩 도중에 "키가 필요합니다"라고 하면, 다음 순서를 따르세요.

1.  AI에게 **"어떤 키가 필요한지, 변수명이 무엇인지"** 물어봅니다. (예: `SLACK_WEBHOOK_URL`, `MY_API_KEY`)
2.  **jun@covering.app에게 요청**합니다: "서버 `/shared/.env`에 `변수명=값` 추가 부탁드려요."
3.  jun@이 추가를 완료하면, AI에게 **"환경변수 추가됐어, 계속 진행해줘"** 라고 말합니다.

## 🚀 배포 흐름 (PR → AI 리뷰 → 자동 배포)

AI에게 "배포해줘" 또는 "올려줘"라고 하면, AI가 직접 main에 올리지 않고 아래 절차를 자동으로 진행합니다.

### 진행 순서

```
AI가 코드 완성
    ↓
브랜치 생성 (feat/날짜-이름)
    ↓
PR(Pull Request) 생성
    ↓
CodeRabbit 자동 코드 리뷰
    ↓
✅ 코멘트 없음 → 담당자 승인 후 main 머지 + 브랜치 삭제 + 자동 배포
💬 코멘트 있음 → 미해결 대화 해결 전까지 머지 차단
```

### 배포 후 확인하기

1.  **Slack 알림 확인**: `#개발팀_커버링랩스` 채널에 배포 완료 메시지가 왔는지 확인합니다.
2.  **주소 접속**: `https://labs.covering.app/[앱이름]` 주소로 접속합니다. (**AWS Client VPN `covering-vpn-v2` 연결 필수** — 2026-04-17부터)
3.  **CodeRabbit 리뷰 결과 확인**: PR 페이지에서 CodeRabbit 코드 리뷰 결과를 확인하세요. (GitHub Actions 로그는 배포 상황을 보여줍니다)
4.  **에러 발생 시**: 화면이 안 나오거나 슬랙 알림이 'Failed'로 오면, [GitHub Actions 로그](https://github.com/covering-app/covering-labs/actions)를 열어 실패한 단계의 오류 내용을 복사한 뒤, **채팅창에 붙여넣어 AI에게 수정을 요청**하세요.

## 🧹 배포 전에 불필요한 파일 거르기

배포 직전에 정책 문서, 방법론 메모, 로그, CSV, 임시 스크린샷이 섞여 올라가는 실수를 막고 싶다면
`release-file-guard`를 설치하세요.

- 설치 문서: [10_출고_전_파일_필터_설치.md](10_출고_전_파일_필터_설치.md)
- 가장 쉬운 설치 명령: `bash .claude/skills/release-file-guard/install-global.sh`
- 설치 후에는 `배포해줘`, `git push`, `gh pr create` 직전에 자동 경고가 뜹니다.

## 🔄 앱 관리 (수정 및 삭제)

- **수정**: "이미 만들어진 [앱이름]에서 이런 기능을 추가해줘"라고 요청하세요.
- **삭제**: "이제 [앱이름]은 필요 없으니 삭제해줘"라고 요청하면 AI가 알아서 서버에서 제거합니다.
