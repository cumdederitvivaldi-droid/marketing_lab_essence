# 런치 — 톤 규칙 + 자가 검수

> 런치 AI 의 가장 큰 차별점 — **마크다운·이모지 금지, 비즈니스 톤 강제**.
> 위치: `lib/ai/lunch-prompt.ts` 의 "기본 규칙" 섹션 + `lunch-policy.md`.

## 톤 규칙 (방문수거와 다른 점)

| 항목 | 방문수거 | 런치 |
|---|---|---|
| 톤 | 친근 + 정중 | **비즈니스 + 간결** |
| 마크다운 | 허용 | **금지** |
| 이모지 | 0~2개 허용 (😊, 🎉 등) | **금지** (`:)` 만 허용) |
| 인사말 | "고객님, 안녕하세요" + 이름 호칭 가능 | **"안녕하세요 커버링 런치입니다 :)" 고정** + 벤더명 호칭 금지 |
| 응답 길이 | 비교적 자유 | 짧고 명확 우선 |

## 왜 다른가

런치는 **B2B (도시락 매장 사장님)** 응대 — 바쁜 사업자가 빠르게 정보 확인. 캐주얼한 이모지 / 마크다운 강조는 부적절. 카카오톡 메시지에 그대로 출력되어야 하므로 마크다운 금지.

## 핵심 규칙 (system prompt 인용)

### 마크다운 금지

> 마크다운 사용 금지 (`**`, `*`, `#`, `-`, 코드블록 등 절대 사용하지 마세요).
> **순수 텍스트만**.

위반 예시:
- `**굵게**` → "굵게" (마크다운 그대로 표시됨)
- `# 헤딩` → "# 헤딩"
- `- 항목 1` → "- 항목 1"
- ` ``` 코드 ``` ` → "```코드```"

올바른 예시:
- 강조 → 줄바꿈 + 들여쓰기 또는 콜론
- 목록 → "- " 가 아닌 "・" 또는 그냥 줄바꿈

### 이모지 금지

> 이모지 사용 금지. 단, **`:)` 텍스트 이모티콘**은 사용 가능.

위반 예시:
- 😊 🎉 ✅ 📦 📅 등 모든 unicode 이모지

올바른 예시:
- `:)` (스마일)
- `:(` (슬픔, 거의 사용 안 함)

### 벤더명 호칭 금지

> 인사말에 벤더명(상호명) 부르지 마세요. 벤더명은 고객 개인명이 아니라 지점/가게 이름입니다.
> 첫 인사는 "안녕하세요 커버링 런치입니다 :)" 로 고정.
> "[상호명]님" 같은 호칭 사용 금지.

위반 예시:
- "카페 ABC 강남점님, 안녕하세요" — 어색
- "도시락마을님 안녕하세요" — 부자연

올바른 예시:
- "안녕하세요 커버링 런치입니다 :)"
- "사장님, 확인드릴 사항이 있어 연락드립니다"

### 정중한 비즈니스 톤

> 친근하고 **간결한** 비즈니스 톤으로 답변하세요.

| 좋은 예 | 나쁜 예 |
|---|---|
| "수거 접수 확인드립니다 :)" | "수거 접수 완료요 ㅎㅎ" |
| "감사합니다 :)" | "감사용 ㅎㅋ" |
| "수량 변경 시 미리 말씀 부탁드립니다!" | "수량 바뀌면 말해주세요~" |

## 자가 검수 (응답 형식 강제)

system prompt 가 위 규칙을 명시 → AI 가 응답 작성 시 self-check.

### 작동 원리
Sonnet 은 system prompt 의 "X 금지" 패턴을 비교적 잘 따름. 그러나:
- 강한 사용자 메시지 (이모지 가득) 에 영향받아 같이 사용할 가능성
- 긴 답변 작성 중 마크다운 자동 사용

### 100% 보장 안 됨
실제 운영에서 가끔 위반 발생:
- "**오늘 5시까지** 입금 부탁드립니다" — `**` 사용
- "수거 접수 확인 ✅" — 이모지 사용

### 보강 옵션 (현재 미구현)
1. **발송 직전 sanitizer** — `app/api/lunch/conversations/[sessionId]/send/route.ts` 에서 발송 직전 마크다운 / 이모지 자동 제거
2. **재생성 트리거** — 응답에 위반 패턴 발견 시 AI 에게 재요청 (latency 증가)
3. **A/B 비교 도구** — 위반 빈도 모니터링

## 정책 문서 (`lunch-policy.md`)

빌드 타임 로드. 159줄. 핵심 섹션:

### 1. 서비스 개요
- 단체 도시락 폐기물 수거
- 서울 / 경기 / 인천 가능
- 정기 / 단발 모두 처리

### 2. 운영 시간
- 주간: 오전 / 오후 (시간 지정 가능)
- 야간: 오후 10시 ~ 오전 6시 (시간 지정 불가)

### 3. 가격 정책
- 수거요금 (지역별)
- 처리요금 (개수 기반)
- 야간 수거 특칙

### 4. 결제 방법
- 링크페이 (카드 / 가상계좌)
- 계좌이체 (사업자)
- 월말정산 (벤더 별 협의)
- 세금계산서 (사업자, Bolta 발행)

### 5. 정산 / 발행
- 정기 수거 월말 정산
- 세금계산서 발행 사이클

### 6. 미수거 / 클레임
- 누락 신고 → NEED_HUMAN
- 환불 / 보상 → NEED_HUMAN

### 7. 행사 수거
- 대량 수거 → NEED_HUMAN (별도 견적)

수정 시 빌드 후 배포 필수 (빌드 타임 로드).

## 자주 깨지는 케이스

### 1. 이모지 출력
- 증상: AI 응답에 😊 또는 ✅ 포함
- 원인: AI 자가 검수 우회 (드물게)
- 해결: system prompt 강조 + 발송 직전 sanitizer 추가 검토

### 2. 마크다운 출력
- 증상: `**`, `#` 등 카카오톡에 그대로 표시
- 원인: AI가 강조 표현으로 자동 사용
- 해결: 위와 동일

### 3. 벤더명 호칭
- 증상: "카페 ABC 강남점님, 내일 수거 확인드립니다"
- 원인: 일반적 친근한 인사 패턴 학습
- 해결: system prompt 의 "벤더명 호칭 금지" 강조 + negative example 추가

### 4. 너무 캐주얼
- 증상: "ㅎㅎ", "~요", "ㅋㅋ" 등 친근한 어미
- 원인: AI가 친근감 표현
- 해결: "비즈니스 톤" 강조 + 좋은 예 / 나쁜 예 추가

### 5. 너무 길어짐
- 증상: 한 응답에 5문단+
- 원인: AI가 친절하게 모든 정보 안내
- 해결: "간결한" 강조 + 응답 길이 가이드라인 추가

## 검증 SQL

```sql
-- 마크다운 사용 의심 (이번 주 런치 ai_auto)
SELECT id, sent_by, content
FROM lunch_messages
WHERE role = 'assistant'
  AND reply_kind = 'ai_auto'
  AND created_at > NOW() - INTERVAL '7 days'
  AND (content LIKE '%**%' OR content LIKE '%# %' OR content LIKE '%```%')
LIMIT 50;

-- 이모지 사용 의심 (basic emoji 패턴 — 완벽한 검출은 어려움)
SELECT id, sent_by, content
FROM lunch_messages
WHERE role = 'assistant'
  AND reply_kind = 'ai_auto'
  AND created_at > NOW() - INTERVAL '7 days'
  AND content ~ '[\u{1F300}-\u{1F9FF}]'  -- emoji 범위 (PostgreSQL 14+)
LIMIT 50;

-- 벤더명 호칭 의심
SELECT lc.vendor_name, lm.content
FROM lunch_messages lm
JOIN lunch_conversations lc ON lc.session_id = lm.session_id
WHERE lm.role = 'assistant'
  AND lm.reply_kind = 'ai_auto'
  AND lm.created_at > NOW() - INTERVAL '7 days'
  AND lm.content LIKE lc.vendor_name || '%님%'
LIMIT 50;
```

- 변경 사항 운영팀 사전 공유 (사장님 응대 톤 변화 가능)
