# 02 — UI 페이지 + 컴포넌트

## 페이지 구조

런치는 **단일 페이지 + 3 탭** 구조. 각 탭이 독립 뷰.

```
/lunch (app/lunch/page.tsx — 2298줄)
├── 탭 1. Orders (주문 관리)        ← LunchOrdersView (잠재적 추출)
├── 탭 2. Invoices (세금계산서)     ← LunchInvoicesView
└── 탭 3. Chat (채팅 응대)          ← LunchChatView
```

`app/lunch/page.tsx` 가 orchestrator + 두 뷰는 부분 추출됨, 일부는 페이지 내부에 잔존 (H5 — 분할 진행 중).

## 탭 1. Orders (주문 관리)

운영팀의 메인 화면. 일별 수거 건 입력·배차·정산 상태 추적.

### 핵심 기능
- 날짜·벤더·상태 필터
- 신규 주문 등록 모달 (수동 입력 or 채팅에서 `<order_data>` 자동 파싱 후 자동채움)
- 인라인 편집 (시간·인원·금액)
- 배차 입력 (driver_name, driver_phone)
- 결제 링크 발송 / 재발송
- 일괄 작업 (체크박스 → 일괄 결제, 일괄 status 변경)

## 탭 2. Invoices (세금계산서)

월별 발행 이력 + 신규 발행.

### 컴포넌트
- `components/lunch/LunchInvoicesView.tsx` — 메인 뷰
- `components/lunch/TaxInvoiceSection.tsx` — 발행 섹션
- `components/lunch/modals/*` — 상세·취소 모달

### 핵심 기능
- 벤더별 발행 이력
- 발행 가능 후보 표시 (해당 월의 미발행 lunch_orders 합계)
- Bolta 발행 (단건 / 월별 통합)
- 발행 취소 (issued → cancelled)
- 발행 실패 사유 표시 (`error_message`)

## 탭 3. Chat (채팅 응대)

방문수거 `/conversations` 와 비슷한 3열 레이아웃. 단일 컴포넌트 [`components/lunch/LunchChatView.tsx`](../../../components/lunch/LunchChatView.tsx).

```
┌─ 좌 (300px) ─────────────┬─ 중앙 ─────────────┬─ 우 (380px) ─────────────┐
│ 세션 목록                  │ 메시지 + AI draft  │ 벤더 패널                 │
│ - status 태그              │ - 매크로 단축      │  - vendor 정보            │
│ - 미읽음 뱃지              │ - 이미지/파일      │  - 최근 주문 N건          │
│ - 검색                     │ - InvalidSession   │  - 정산 사이클            │
│                            │   자동 종료        │                          │
└────────────────────────────┴───────────────────┴──────────────────────────┘
```

### 특이점
- **`<order_data>` 자동 파싱**: AI 가 응답에 `<order_data>{...}</order_data>` 태그 포함 시, 클라이언트가 파싱해 "주문 등록" 버튼 활성화 → Orders 탭의 신규 주문 모달 자동채움
- **status="closed" 잠금**: 세션 종료 시 textarea/버튼/이미지 발송 모두 비활성 (사용자 메모 반영)
- **InvalidSession (-502) 자동 종료**: send/route.ts 가 감지 → status closed 전환

## 페이지 ↔ API 매핑

| 탭 | 주요 API |
|---|---|
| Orders | `/api/lunch/route.ts` (목록), `/api/lunch/payment*`, NicePay 라우트 |
| Invoices | `/api/lunch/invoices/*`, Bolta API (간접) |
| Chat | `/api/lunch/conversations/*` |

## 매크로 / 가이드 / 자동 응답

방문수거에 비해 가벼움:
- 매크로: 공유 `macros` 테이블 사용 (방문수거와 같은 테이블, 카테고리만 분리 가능)
- 가이드 발송: 별도 라우트 없음 (필요 시 매크로로 처리)
- 자동 응답: AI Phase `idle/inquiry` 가 처리

## 상담사 영역

- `/lunch` 의 "채팅" 탭에서 응대
- 매크로·이미지·파일 발송 가능
- AI draft 검토 후 발송 (방문수거와 동일 패턴)
- 송신본 자동 분류 → `lunch_messages.reply_kind` 누적
