## 메모 시스템 (셀 단위 토론)

대시보드의 모든 셀(KR / Journey 의 phase / Health 의 metric / Traffic 의 global) 에 메모 추가 가능.

### cell_key 컨벤션
- KR: `kr1`, `kr2`, `kr3`
- Journey: `<phase>:<metric>` (예: `phase_4:conversion`, `phase_2:churn_keyword`)
- Health: `<metric>` (예: `no_pickup`, `cancel`, `nps`)
- Traffic: `global`

### API
- `GET /api/new_dashboard/notes?section=...` — 조회 (summary 모드는 셀별 카운트만)
- `POST /api/new_dashboard/notes` — 작성
- `PATCH /api/new_dashboard/notes/[id]` — 수정 / 해결 처리
- `DELETE /api/new_dashboard/notes/[id]` — 삭제 (작성자 본인만)

### 표시
- 각 셀에 메모 배지 (개수·미해결 우선)
- 클릭 → 슬라이드오버에서 메모 누적·작성·해결

## CS Realtime 카드 동작

자세히는 [`08-gotchas.md`](08-gotchas.md). 핵심:

1. 페이지 진입 → `/api/new_dashboard/cs-realtime` 10초 polling
2. 응답: `operators[]` (이름·시스템·오늘 답변 수·근무시간·시스템별 분리·lastReplyAt)
3. 동시에 `useCsRealtimePresenceContext` 로 viewers (presence 채널) 수신
4. 카드 status 결정:
   - presence 있음 → presenceLevelOf (5분/15분 임계값)
   - presence 없음 + lastReplyAt < 5분 → "online (외부 도구 답변)"
   - presence 없음 + lastReplyAt 없음 → "offline"

### 카드 정렬
1차: 상태 (online > idle > away > offline)
2차: 오늘 답변 많은 순
3차: 근무시간 긴 순

### WorkHistoryModal
카드 클릭 → 일별 근무 기록 모달:
- 880×720 드래그 가능
- 7일 / 30일 / 90일 프리셋 + 사용자 정의 날짜 범위
- 7-컬럼 테이블 (날짜·방문 상담/답변·런치 상담/답변·채널 상담/답변·근무 분)
- API: `/api/new_dashboard/cs-realtime/work-history?counselor=X&days=N`

## 모달들

- `ComplaintModal` — 불만 row 클릭 → 상세 + "불만 아님" 처리 (분류 → `none`)
