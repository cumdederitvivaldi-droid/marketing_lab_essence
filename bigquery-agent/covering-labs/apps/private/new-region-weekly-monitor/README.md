# 신규 지역 주간 모니터

신규 지역 운영 권역의 주간 성장 상태를 Slack으로 요약 발송하는 배치입니다.

## 목적

신규 지역 서비스 권역별 MAU(Monthly Active User) 500명 도달 추이를 추적하고 주간 성장률, 신규 결제자, 예상 도달 시기를 권역별로 분석하여 매주 월요일 제품팀_data Slack 채널로 발송합니다. 권역별 진행 상태(통과권/근접권/보류권)와 데이터 리스크를 함께 전달합니다.

## 실행 환경

- **실행 방식**: Cron 배치
- **실행 주기**: 매주 월요일 10:00 KST (cron: `0 10 * * 1`)
- **실행 서버**: GCP VM (crontab — SA 계정으로 실행)

## 주요 파일

| 파일 | 역할 |
|---|---|
| `src/main.py` | 메인 실행 파일. BigQuery 쿼리 실행, 데이터 수집, 메시지 생성 및 Slack 발송 |
| `src/weekly_metrics.sql` | 권역별 주간 지표 조회. MAU, 전주 대비 변화율, 신규 결제자, 500까지 필요한 사용자 수, 예상 도달 날짜 계산 |
| `src/unmapped_regions.sql` | 권역 매핑 밖의 주문을 집계. 중부권역(대전/세종/충북/충남)에서 매핑되지 않은 사용자의 30일 MAU |
| `deploy.yml` | 배치 배포 설정 파일 |
| `requirements.txt` | 파이썬 의존성 (현재 표준 라이브러리만 사용) |

## 환경변수

| 환경변수 | 필수 | 기본값 | 설명 |
|---|---|---|---|
| `SLACK_BOT_TOKEN` | 필수 | - | Slack 봇 토큰. `chat:write` 권한 필요 |
| `NEW_REGION_DASHBOARD_URL` | 선택 | Grafana 신규 지역 대시보드 | Slack 메시지에 첨부될 대시보드 링크 |
| `BQ_BIN` | 선택 | `bq` (PATH) 또는 `~/google-cloud-sdk/bin/bq` | BigQuery CLI 실행 경로 |
| `ENV_FILE` | 선택 | `/shared/.env` | 환경 파일 경로. Crontab 실행 시 자동 로드 |

Slack 루트 메시지는 `[충청도권 신규 지역 500 MAU 달성률]` 제목으로 `#제품팀_data`에 발송하고, 상세 본문은 같은 메시지의 스레드 답글로 쌓습니다.
첫 실행 때 만든 루트 메시지의 `thread_ts`는 `logs/thread_state.json`에 저장해 다음 실행부터 같은 스레드를 재사용합니다. 필요하면 `NEW_REGION_WEEKLY_MONITOR_THREAD_TS` 환경변수로 고정 스레드를 직접 지정할 수 있습니다.

## 실행 방법


### 실제 실행 (Slack 발송)

```bash
python3 src/main.py
```

## 분석 기준

권역을 다음 3가지로 분류합니다.

| 분류 | 기준 | 아이콘 |
|---|---|---|
| 통과권 | 최근 30일 MAU 500 이상 | ✅ |
| 근접권 | 현재 속도로 90일 안에 MAU 500 도달 예상 | 🟡 |
| 보류권 | 그 외 | 🔴 |

ETA 계산: `권역 오픈 이후 경과일수 / (현재 MAU / 경과일수) = 총 필요 일수` → 오픈일 + 총 필요 일수

권역 구분은 코드의 SQL에서 아래처럼 명시합니다.

- 천안: `천안시 동남구`, `천안시 서북구`
- 아산: `아산시`
- 대전 유성: `유성구`
- 대전 서중: `서구`, `중구`
- 대전 대덕동: `대덕구`, `동구`
- 세종: `세종특별자치시`
- 청주 상당서원: `청주시 상당구`, `청주시 서원구`
- 청주 흥덕: `청주시 흥덕구`
- 청주 청원: `청주시 청원구`

## 의존 서비스

- **BigQuery**: 주문, 사용자, 서비스 지역 데이터 조회
  - 데이터셋: `covering-app-ccd23.secure_dataset`
  - 테이블: `service_region`, `order_v2`, `order_line`, `order_address_snapshot`, `product`
- **Slack API**: `chat.postMessage` (메시지 발송)
- **Grafana**: 대시보드 링크 (메시지에 포함)

## 주의사항

- **데이터 리스크**: SQL에 명시한 권역 밖의 대전/세종/청주/천안/아산 주문이 있으면 Unmapped Regions 섹션으로 따로 추적합니다.
- **타임존**: 모든 날짜 계산은 Asia/Seoul(KST) 기준입니다.
- **BigQuery 타임아웃**: 쿼리 타임아웃은 180초로 설정되어 있습니다.
- **Slack 발송 실패**: 네트워크 오류나 토큰 만료 시 전체 배치가 실패합니다.
- **MAU 기준**: 결제 완료된 서비스 주문을 한 고유 사용자만 집계합니다.
