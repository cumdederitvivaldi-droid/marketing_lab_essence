# yagan-rider-gap-alert — 야간기사 완료 간격 30분 초과 알림 배치

## 목적

야간기사가 수거/봉투배송 성공·실패 이후 30분이 지나도록 다음 완료 건이 없으면
`#운영_야간수거` 채널에 해당 기사 이름을 자동 알림.
운영팀이 실시간으로 진행이 멈춘 기사를 파악하고 즉시 대응할 수 있도록 한다.

## 실행 시점

- **크론**: `*/5 22-23,0-7 * * *` (매 5분, 22:00~07:55 KST)
- **추적 구간**: 22:00 ~ 다음날 08:00 (야간 세션)
- **상태 리셋**: 매일 22:00에 새 야간 세션 시작 (이전 알림 기록 초기화)

## 감지 조건

1. 오늘 배차된 기사 (`fulfillment.rider_id IS NOT NULL`)
2. 오늘 COMPLETED 또는 FAILED 완료 건이 1개 이상 (수거·봉투배송 모두 포함)
3. 마지막 완료 시각으로부터 30분 이상 경과 (`TIMESTAMP_DIFF >= 30`)
4. 아직 미완료(`COMPLETED/FAILED/CANCELED` 외) fulfillment가 남아 있음
5. 해당 조건으로 아직 알림을 보내지 않은 기사만 (중복 알림 방지)

## 상태 관리

- 파일: `state/alert_state.json`
- 세션 키: 야간 세션 시작 날짜 (`session_date`)
  - 22:00~23:59 → 당일 날짜
  - 00:00~07:59 → 전날 날짜
- 구조:
  ```json
  {
    "session_date": "2026-04-30",
    "alerted": {
      "rider_id_123": "22:15"
    }
  }
  ```
- 동일 rider_id의 `last_completed_time`이 변경되면(새 완료 발생 후 또 멈추면) 재알림

## Slack 메시지 형식

```text
⏰ 야간기사 수거 간격 알림 — 2026-04-30 22:50 기준

마지막 완료 후 30분 이상 다음 완료 없는 기사님: *2명*

🛵 *해당 기사님 목록*
• 홍길동 — 마지막 완료 22:15 (35분 전) / 잔여 3건
• 김영희 — 마지막 완료 22:10 (40분 전) / 잔여 2건
```

## 파일 구성

| 파일 | 역할 |
|---|---|
| `deploy.yml` | cron `*/5 22-23,0-7 * * *`, type: batch |
| `src/main.py` | 진입점 — 조회·상태관리·알림 오케스트레이션 |
| `src/config.py` | 환경변수 로드 |
| `src/bq_client.py` | BigQuery 조회 |
| `src/query.sql` | 30분 이상 간격 기사 추출 쿼리 |
| `src/slack_client.py` | Slack 알림 전송 |
| `src/state_manager.py` | 야간 세션 상태 관리 및 중복 알림 방지 |
