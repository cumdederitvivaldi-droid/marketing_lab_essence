# 환경변수 가이드

covering-labs 전체 앱에서 공통으로 사용 가능한 환경변수 목록과 신규 변수 추가 절차를 정리합니다.

> **단일 진실의 원천**: [`apps/AGENTS.md` — 환경변수 레지스트리](../apps/AGENTS.md) 섹션이 항상 최신 목록입니다. 이 문서는 공통 변수 개요 및 탐색 가이드입니다.

---

## 저장 위치 요약

| 위치 | 설명 | 대상 |
|---|---|---|
| `/shared/.env` (private VM) | 모든 batch·nextjs·nestjs 앱 공유. 배포 스크립트가 자동 source | private VM 앱 전체 |
| `/shared/.env` (public VM) | covering-spot, covering-talk 등 공유 | public VM 앱 |
| `/shared/apps/[앱이름]/.env` | 앱별 격리 (같은 키명이지만 다른 값이 필요할 때) | 격리 필요 앱 |
| 앱 디렉토리 `.env.example` | 키 목록 문서화. **실제 값 커밋 금지** | 앱별 |

batch 앱은 `config.py`의 `_load_env_file()` 패턴으로 `/shared/.env`를 자동 로드합니다.

## private VM 공통 변수

> 이미 VM에 설정되어 있습니다. 새 앱에서 별도 값 추가 없이 코드에서 바로 참조하세요.

| 변수명 | 용도 | 비고 |
|---|---|---|
| `SLACK_BOT_TOKEN` | covani-pickup 봇 토큰 (채널 메시지·DM·파일 업로드) | 15개+ 앱 공유. 채널 ID는 앱별로 등록 |
| `FLARELANE_PROJECT_ID` | FlareLane 프로젝트 ID | flarelane 관련 앱 공유 |
| `FLARELANE_API_KEY` | FlareLane API 키 (이벤트 트래킹) | flarelane 관련 앱 공유 |
| `GCP_PROJECT` | GCP 프로젝트 ID (`covering-app-ccd23`) | VM 서비스 계정 자동 인증 — 키 파일 불필요 |
| `AIRBRIDGE_TOKEN` | Airbridge API 인증 토큰 | airbridge-ads-cost-sync, covering-invite |
| `AIRBRIDGE_APP` | Airbridge 앱 식별자 (기본값: `coveringprod`) | |

## 충돌 주의 변수

같은 키명이지만 앱마다 다른 값이 필요한 경우. `/shared/.env`는 단일 값만 저장하므로 앱별 `.env`로 격리하세요.

| 변수명 | 충돌 상황 | 해결 방법 |
|---|---|---|
| `SLACK_BOT_TOKEN` | covering-talk은 전용 봇 토큰 사용 | `/shared/apps/covering-talk/.env`에 별도 주입 |
| `DHERO_TOKEN` | 앱마다 다른 spot 토큰 | 앱별 `.env`에 격리 |
| `DHERO_SPOT_CODE` | 앱마다 다른 spot 코드 | 앱별 `.env`에 격리 |
| `CHANNELTALK_ACCESS_KEY/SECRET` | 앱마다 다른 채널톡 계정 가능성 | 사용 전 공유 가능 계정인지 확인 |

## 신규 환경변수 추가 절차

> ⚠️ **코드 작성 전에 레지스트리를 먼저 확인하세요.** 훅(`env-var-registry-check.py`)이 미등록 변수를 감지하면 자동 경고합니다.

```text
1. apps/AGENTS.md 레지스트리에서 동일/유사 용도 변수 확인
   └─ 있으면 → 재사용 (새 이름 생성 금지)
   └─ 없으면 → 2번으로

2. apps/AGENTS.md 레지스트리 테이블에 변수명 먼저 추가

3. VM에서 직접 값 추가
   sudo -u sa_109369409955768144646 nano /shared/.env

4. 앱의 .env.example에도 변수명 + 용도 주석 추가

5. 코드에서 사용

6. PR 설명에 추가한 환경변수 내용 명시
```
