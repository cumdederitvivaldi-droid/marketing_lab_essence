## 수집되는 로그

| 항목 | 예시 |
|---|---|
| SSH 접속/종료 | 누가, 언제, 어디서(IP) 접속 |
| crontab 실행 | 누가, 언제, 무슨 명령 실행 |
| sudo 사용 | 누가, 무슨 명령을 root로 실행 |
| 시스템 서비스 | 서비스 시작/중지/에러 |
| Cloud Storage | 버킷 파일 업로드/다운로드/삭제 |
| GCP 관리 작업 | IAM 변경, 인스턴스 시작/중지 |

## 로그 보존 기간

| 종류 | 보존 | 비고 |
|---|---|---|
| syslog, auth, cron | **30일** | Cloud Logging `_Default` 버킷 |
| GCP 감사 로그 (IAM 변경 등) | **400일** | `_Required` 버킷 (삭제 불가) |
| 서버 `/var/log` | 7~14일 | logrotate 기반 |

| 알림 | 조건 | 지속 시간 |
|---|---|---|
| CPU 사용률 위험 | 80% 이상 | 5분 |
| 메모리 사용률 위험 | 85% 이상 | 5분 |
| 디스크 사용률 위험 | 80% 이상 | 5분 |
| 서버 응답 없음 | CPU 메트릭 수집 안 됨 | 5분 |

**알림 시 대응:**

| 알림 | 확인 명령어 |
|---|---|
| CPU | `top` 또는 `htop` |
| 메모리 | `free -h` |
| 디스크 | `df -h` / `sudo du -sh /home/*` |
| 서버 다운 | `gcloud compute instances list` → 시작 |

알림 정책 관리: https://console.cloud.google.com/monitoring/alerting?project=covering-app-ccd23

### 로그 탐색기 (직접 쿼리)

https://console.cloud.google.com/logs/query?project=covering-app-ccd23

**SSH 접속 기록:**
```
resource.type="gce_instance"
logName:"auth"
jsonPayload.message:"Accepted"
```

**crontab 실행:**
```
resource.type="gce_instance"
logName:"syslog"
jsonPayload.message:"CRON"
```

**sudo 사용:**
```
resource.type="gce_instance"
logName:"auth"
jsonPayload.message:"COMMAND"
```

**특정 사용자 (예: beige):**
```
resource.type="gce_instance"
jsonPayload.message:"beige"
```

**Cloud Storage 파일 작업:**
```
resource.type="gcs_bucket"
resource.labels.bucket_name="covering-labs"
```

**GCP 관리 감사:**
```
logName:"cloudaudit.googleapis.com%2Factivity"
```

## 서버에서 직접 로그 보기 (sudo 필요)

```bash
# SSH 접속 기록
sudo grep 'Accepted' /var/log/auth.log | tail -20

# 특정 사용자
sudo grep 'beige' /var/log/auth.log | tail -10

# 실시간 모니터링
sudo tail -f /var/log/auth.log | grep 'Accepted'

# 전체 crontab 목록
for user in $(cat /etc/passwd | grep '/home' | cut -d: -f1); do
  cron=$(sudo crontab -u $user -l 2>/dev/null)
  if [ -n "$cron" ] && ! echo "$cron" | grep -q 'no crontab'; then
    echo "[$user]"
    echo "$cron"
    echo ""
  fi
done

# cron 실행 기록
sudo grep CRON /var/log/syslog | tail -20

# sudo 사용 기록
sudo grep 'COMMAND' /var/log/auth.log | tail -20

# 디스크 사용량
df -h /
sudo du -sh /home/*/
sudo find /home -size +100M -exec ls -lh {} \;
```

## 로깅 환경 구성

| 구성요소 | 상태 |
|---|---|
| Google Cloud Ops Agent | v2.64.0 |
| 수집 로그 | `/var/log/syslog` + `/var/log/auth.log` |
| 메트릭 | CPU, 메모리, 디스크, 네트워크 |
| GCS 감사 | Data Access 로그 활성화됨 (`ADMIN_READ`, `DATA_READ`, `DATA_WRITE`) |
| 설정 파일 | `/etc/google-cloud-ops-agent/config.yaml` |
