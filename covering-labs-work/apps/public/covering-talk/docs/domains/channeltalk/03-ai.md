# 03 — AI 파이프라인 (분류 → RAG → 생성 → 톤)

## 4단계 파이프라인

```
[고객 메시지]
   ↓
1. 분류 (Sonnet)
   - 84+ 카테고리 중 1~3개 선택
   - 재방문 여부 판별 (Stage 0 — continuing / new / combined)
   ↓
2. RAG 병렬 로드 (Voyage AI 임베딩)
   - 정책 문서 매칭 섹션 (consultation_embeddings)
   - 매크로 후보 (macro_embeddings 172건)
   - 과거 Q&A 페어 (consultation_embeddings)
   - 서비스 지역 검증 (service_areas)
   ↓
3. 답변 생성 (Sonnet)
   - 카테고리별 prompt_rules (category_prompts.prompt_rules)
   - RAG 인용
   - 백오피스 정보 inline (있으면)
   ↓
4. 톤 다듬기 (Haiku) [선택]
   - 친근한 톤·이모지·존댓말 보정
   ↓
[SuggestPanel 에 표시]
```

**핵심 파일:**
- [`lib/channeltalk-ai/suggest.ts`](../../../lib/channeltalk-ai/suggest.ts) — 전체 파이프라인 진입점
- [`lib/channeltalk-ai/normalize.ts`](../../../lib/channeltalk-ai/normalize.ts) — 메시지 정규화 + 분류
- [`lib/channeltalk-ai/category-prompts.ts`](../../../lib/channeltalk-ai/category-prompts.ts) — 카테고리별 프롬프트
- [`lib/channeltalk-ai/service-area.ts`](../../../lib/channeltalk-ai/service-area.ts) — 주소 → 서비스 지역
- [`lib/channeltalk-ai/validate.ts`](../../../lib/channeltalk-ai/validate.ts) — 카테고리별 정책 조회

## 모델

| 단계 | 모델 | 이유 |
|---|---|---|
| 분류 (Stage 0+1) | Sonnet | 정확도 우선 (84 카테고리 중 정확한 1-3 선택) |
| 답변 생성 (Stage 2) | Sonnet | 자연스러운 답변 + RAG 인용 |
| 톤 다듬기 | Haiku | 비용·속도 (자유 보정만) |
| RAG 임베딩 | Voyage AI (`voyage-2`) | 한국어 강세 + 비용 |

`app_settings.ai_provider` 로 Anthropic ↔ OpenAI 런타임 전환 가능 (단 Voyage 는 별개).

### 카테고리별 prompt_rules

각 카테고리마다 `category_prompts.prompt_rules` 에 답변 규칙 저장. 예시:

```
카테고리: 이용_배출품목
prompt_rules:
  - 가능한 품목 목록 명확히 안내
  - 가구·가전·자재는 방문수거(별도 채널) 안내
  - 도시락 폐기물은 런치 서비스 안내
  - 이모지 1개 이내
정책 섹션: ["배출품목", "분리수거"]
```

운영팀이 콘솔에서 직접 수정 가능 (`/api/settings/category-prompts`). 변경 즉시 다음 호출부터 반영.

## RAG 인용 후보

| 소스 | 테이블 | 생성 방법 |
|---|---|---|
| 정책 문서 | `consultation_embeddings` | `tools/channeltalk-ai/embed-consultations.ts` 가 `policy-document.md` 를 청크 단위 임베딩 |
| 매크로 | `macro_embeddings` | `tools/channeltalk-ai/embed-macros.ts` |
| 과거 Q&A | `consultation_embeddings` | `consultation-pairs-classified.json` 임베딩 |
| 서비스 지역 | `service_areas` | (m006) 행정동 마스터 |

코사인 유사도 top-K (보통 K=5) 선택 후 답변 생성에 inline.

## 백오피스 정보 inline

채널톡 응대 중 고객 정보가 필요한 카테고리 (예: "내 주문이 어떻게 됐나요?"):

1. 클라이언트 (`/channeltalk`) 가 메시지에서 phone 추출
2. `/api/backoffice/lookup` POST → `backoffice_requests` INSERT
3. 스크래퍼가 처리 → result 채움
4. 클라이언트 polling 으로 수신
5. AI suggest 호출 시 `backofficeData` 파라미터로 전달
6. `category-prompts.ts` 가 백오피스 정보를 prompt 에 inline

## 분류 (reply_kind) — 사후 통계

상담사가 추천 답변 발송 시 `lib/utils/reply-classify.ts`:
- `overlap ≥ 1.0` → `ai_auto` (그대로 채택)
- `overlap ≥ 0.6` → `ai_assist` (수정)
- 그 외 → `human` (직접 / 매크로)

저장: `channeltalk_reply_logs` (chat_id + manager_name + reply_kind + draft_char_overlap + sent_at).

**ai_draft 보존**: 사용자 메모에서 — "채택" 버튼 클릭 후에도 draft 가 비교용으로 보존되도록 `lastDraftForCompareRef` 처리.

## 자동 추천 보내기 흐름 ("외부 도구 답변" 보강)

1. 상담사가 채널톡 데스크앱 / 모바일 / web desk.channel.io 로 답변 → 채널톡 메시지 발송
2. 우리 시스템에서 polling 또는 webhook 수신
3. `channeltalk_reply_logs` INSERT (manager_name 기록)
4. **대시보드 카드는 "외부 도구 답변" 으로 표시** (presence 채널엔 없지만 lastReplyAt 5분 내)

## A/B 비교 (AiCompareModal)

운영 중 프롬프트 변경 효과 측정:
- 같은 메시지로 2번 호출 (다른 프롬프트 / 모델)
- A vs B 답변 보여주고 상담사가 선택
- 채택률 통계로 우열 판단

## 디버깅 (SuggestDebugPanel)

파이프라인 단계별 출력:
- Stage 0 결과 (continuing/new/combined)
- Stage 1 결과 (카테고리)
- RAG 매칭 (top-K + 유사도)
- Stage 2 raw 응답 (Sonnet)
- 톤 보정 전후

## 학습 데이터 갱신

자세히는 [`tools/channeltalk-ai/README.md`](../../../tools/channeltalk-ai/README.md):
- 정책 문서 수정 → `embed-consultations.ts` 재실행
- 새 매크로 추가 → `embed-macros.ts`
- 카테고리 추가 → `seed-category-prompts.ts` + `category-labels.ts` 갱신
- 새 서비스 지역 → `seed-service-areas.ts`

## 안전 장치

- 서비스 지역 검증: 미서비스 지역 → 안내 + 답변 차단
- 정책 위반 (가격·약관 잘못 안내) → category_prompts 의 ai_scope_note 로 가드
- 백오피스 데이터 부재 시 → "백오피스 정보 조회 실패, 직접 확인 필요" 명시
- 카테고리 분류 실패 시 → "기타_확인필요" 로 fallback
