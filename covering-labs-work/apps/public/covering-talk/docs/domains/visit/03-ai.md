# 03 — AI 로직 (Phase 머신 + 프롬프트 + 분류)

## 모델

| 용도 | 모델 | 위치 |
|---|---|---|
| 메인 응답 (Phase 1~8) | Anthropic Sonnet | `lib/ai/claude.ts` (provider abstraction `lib/ai/ai-client.ts`) |
| 커바니 (assistant-hint) | Anthropic Haiku | `app/api/conversations/[sessionId]/assistant-hint/route.ts` |

런타임 provider 전환: `app_settings.ai_provider` (`"anthropic"` | `"openai"`).

## Phase 머신 (9 단계)

```
phase_1 (인사)
   ↓
phase_2 (정보수집: 주소·층·엘베·주차·품목)
   ↓
phase_3 (사양확인)
   ↓
phase_3-1 (품목 수정 분기) ──┐
   ↓                          ↓
phase_4 (견적) ←──────────────┘
   ↓
phase_5 (넛지)  ── (auto-nudge cron 10시)
   ↓
phase_6 (예약접수)
   ↓
phase_7 (예약확정) → orders INSERT + sendToCovering
   ↓
phase_8 (사후관리)
   ↓
closed
```

**파일:**
- 정의: [`lib/ai/phases.ts`](../../../lib/ai/phases.ts) — `Phase` enum + 표시명 + 다음 phase 매핑
- 전환 로직: [`lib/ai/phase-transitions.ts`](../../../lib/ai/phase-transitions.ts) — 어떤 신호에 어디로 갈지
- 프롬프트: [`lib/ai/prompt.ts`](../../../lib/ai/prompt.ts) — 시스템 프롬프트 + Phase 별 user 프롬프트 빌더
- 정책: [`lib/ai/pickup-policy.md`](../../../lib/ai/pickup-policy.md) — 운영 규칙 원문 (커바니·프롬프트 공동 소스)

## 프롬프트 구성

```
시스템 프롬프트 = 페르소나 + 톤(존댓말 + ":)") + 기본 정책 요약
                  + 공휴일 환각 방지 (app_settings.abc_capacity.holidays + 2026 fallback)
                  + 서비스 지역 (행정동 기준 — 법정동 아님)

User 프롬프트 = 현재 Phase 컨텍스트
                + collected_info 스냅샷
                + 최근 N개 메시지
                + ci.requestedDate (서버 사전 판정 후 플래그 주입)
```

`prompt-blocks.ts` 가 재사용 블록 (인사·정보수집·사양확인 등) 을 모듈화. Phase 별 prompt builder 가 조합.

## 분류 (reply_kind) — 사후 통계

상담사 답변 발송 후 `lib/utils/reply-classify.ts:classifyReply` 가 자동 호출:

```
overlap = bigram Sørensen-Dice(ai_draft, sent_text)

overlap ≥ 1.0          → ai_auto    (AI draft 그대로)
overlap ≥ 0.6          → ai_assist  (AI draft 일부 채택)
그 외 (또는 draft 없음) → human     (직접 작성)
```

저장: `messages.reply_kind` + `responded_in_ms` + `draft_char_overlap` (m026).

## 견적 (Quote)

`/api/quote/calculate/route.ts` (665줄, 큰 모듈):

1. `collected_info.items` 정규화 → 표시명 매칭
2. `products` 테이블 lookup (단가·부피)
3. `region_prices` lookup (지역 기본가)
4. 사다리차 필요 시 `ladder_fees` 적용
5. 작업인원 산출 → 인건비
6. items_total + base_price + ladder_fee + tax → total
7. `quotes` + `quote_items` INSERT
8. 응답 → 상담사 화면에 견적 카드 + 메시지 자동 발송 후보

품목 검색은 Voyage 임베딩 (`product_embeddings`) + 별칭 매칭 hybrid (`lib/utils/product-search.ts`).

## 품목 추출 (extract-items v1, v2)

- v1: `/api/conversations/[sessionId]/extract-items` — 단순 패턴 매칭
- v2: `/api/conversations/[sessionId]/extract-items-v2` — Sonnet 호출, 모호한 표현 처리

`lib/ai/ambiguous-items.ts` 가 동음이의 품목(예: "냉장고" → 양문/일반/김치냉장고) 분기 처리.

## district 추론

`lib/ai/district-resolver.ts` — 주소 텍스트에서 시·구·동 추출 + 서비스 가능 행정동(`service_areas`) 체크.
- 서비스 지역은 **행정동 기준** (법정동 아님 — 사용자 메모에서 강조)
- 미서비스 지역 → Phase 정지 + 안내 메시지

## 이미지·OCR

- `/api/conversations/[sessionId]/send-image` — 상담사가 이미지 발송
- 고객 이미지 → `messages.image_url` 누적 → 견적 시 참고
- OCR/Vision 자동 분석은 (현재) 없음 — 상담사 수동 검토

## 커바니 (assistant-hint)

- 트리거: 고객 메시지 1.5초 후 (자동) 또는 상담사 수동 호출
- 모델: Haiku (저비용·고속)
- 응답: `{ hint: string, section: string | null }` JSON
- `hint`: 1줄 코칭 (예: "예약일 미정 — 가능한 날짜 3개 제시 권장")
- `section`: 정책 문서 heading slug → 📖 클릭 시 PolicyModal 점프
- 자기 답변 루프 방지: assistant role 이후 호출 안 함

## 안전 장치

- 공휴일 환각 방지: 서버가 holidays 명시 → 목록에 없는 날짜 공휴일 판단 금지
- 서비스 지역 검증: 행정동 마스터 (`service_areas`) 미일치 시 자동 차단
- 품목 매칭 신뢰도: 임베딩 score < 임계값 → "확인 필요" 표시
- 견적 한도: 음수·비현실적 큰 수 가드 (calculate 내부)
