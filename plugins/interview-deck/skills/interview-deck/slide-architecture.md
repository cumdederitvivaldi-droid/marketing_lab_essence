# 슬라이드 아키텍처 — 서사 레시피 & 컴포넌트 카탈로그

면접 발표덱 15건 이상에서 반복 검증된 골격(원작: 이도형). `deck-template.html`의 컴포넌트와 짝지어 쓴다.

## 표준 13p 골격 (그로스/마케팅 직무 기본형)
1. **타이틀** — thesis 한 문장(헤드라인 강조 1곳) + 회사·직무·이름 pill
2. **(선택) 서론/페르소나** — 면접관 교체·맥락 필요 시. "왜 더 힘들까?"식 현장 목소리
3. **직무 정의** — JD에서 뽑은 핵심 책임 N개 (역할 재확인)
4. **회사/제품 소개** — 실측 캡처 1장 + 단계·수치(출처 태그)
5. **페인포인트** — 직접 써보니 발견한 약점 3개 (← 전략 3개와 1:1로 설계)
6. **경쟁사 표** — 직접·간접 경쟁사 강약점 + **공통 공백(빈틈)** 강조행
7. **USP / 전략 개요** — 공백을 메우는 차별점. 페인 3 → 전략 3 매핑 도식
8~10. **전략 1·2·3** — 각: 목업(폰/검색결과/푸시) + **벤치마크 레퍼런스 캡처** + 근거 수치
11. **(선택) BM / 매출모델** — 수수료·구독·리테일미디어 + 정량 프로젝션(가정 명시)
12. **기대효과 / KPI** — 단일 NSM(North Star) + 증분 논리(곱셈 구조 선호)
13. **왜 나 + 지원동기** — JD 자격요건 ↔ 본인 포트폴리오 1:1 매핑 + 휴먼 동기 포부

## 변주
- **컬쳐핏/최종**: 5~7을 "회사 사각지대 진단 + 베팅형 why-me"로 축소·대체
- **과제형**: 과제 문항 답안 슬라이드 우선, 분석덱 분리
- **압축본(면접 직전)**: 개념 먼저→how 흐름으로 중간 상세 삭제(perl 범위삭제로 재넘버 가능, 원본은 git 히스토리 보존)

## 서사 원칙 (재작업 방지)
- **페인 → 전략 1:1**이 척추. 페인 개수 = 전략 개수.
- 경쟁사 표의 **빈칸이 곧 USP**. 빈칸 없으면 각도를 바꿔라.
- 모든 전략 = 외부 **벤치마크 레퍼런스**로 뒷받침(주장 아닌 증거).
- 모든 외부 수치 = `[자사발표]/[제3자]/[추정]/[미검증]` 태그 + footnote 출처.
- 빌드 전 **목차를 사용자에게 확인**받는다.

## 컴포넌트 카탈로그 (deck-template.html 클래스)
- 텍스트: `h1/h2/h3`, `.lead`, `.kicker`(.vio/.teal/.amber/.rose), `.hl`(형광펜 강조, 슬라이드당 1곳), `.goal`(목표 pill)
- 박스: `.card`(.blue/.vio/.teal/.amber/.rose/.out/.dark), `.callout`(검정), `.quote`(그라디언트 인용)
- 데이터: `.metric .big/.lab`(KPI 숫자), `table`(`tr.hi` 강조행, `.chk`/`.x` O·X), `.flow .step .arr`(퍼널)
- 칩: `.tag`, `.pill`(.k/.blue/.vio/.teal)
- 캡처: `.shot`(이미지), `.shotwrap .badge`(라벨 핀), `.cap`(캡션)
- 광고 목업: `.adcard`(검색광고), `.stepbox`(단계카드)
- 출처: `.foot .src`(하단 출처바, `b`로 라벨), `.flag`(이탤릭 단서)
- 리스트: `ul.clean`(.vio/.teal/.rose 색 불릿)

## 톤 오버라이드 방법
`:root`의 액센트 토큰(`--blue`,`--blue-d`,`--blue-soft`,`--violet*` 등)을 회사 브랜드 컬러로 교체. 구조 토큰(`--ink`,`--bg`,`--soft`,`--line`,`--radius`,`--shadow`,`--font`)은 유지. 폰트는 Pretendard 고정. 예시: 채널톡=블루#1B66FF+바이올렛#6E4BFF, 리웨이크=네이비#15203A+카멜#B08B57, 모요=인디고+레드.
