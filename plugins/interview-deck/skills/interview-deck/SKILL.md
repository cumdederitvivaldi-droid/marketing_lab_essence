# Interview Deck — 면접 발표덱 제작 파이프라인

면접 발표덱 15건 이상에서 반복 검증된 6단계 워크플로우를 병렬화·표준화한 스킬이다.
목표: **리서치를 병렬로 돌려 시간을 줄이고, 검증된 디자인 시스템으로 품질을 고정하고, 배포·기록을 자동화**한다.

가장 먼저 같은 폴더의 `slide-architecture.md`(서사 레시피 + 컴포넌트 카탈로그)와 `deck-template.html`(빌드 스캐폴드)을 읽어라.

## 0. 인테이크 (없으면 AskUserQuestion으로 한 번에)
- 회사/직무, 채용공고 URL(원티드 등), 면접 단계(1차/컬쳐핏/과제 등)
- 분량(기본 13p), 강조하고 싶은 thesis/서사 방향
- 면접 과제 문항이 있으면 원문 그대로
- 새 데이터/제약(내부데이터 공개 금지 여부 등)
- 발표자 이름(덱 타이틀의 `{{이름}}` 자리)
이미 진행하던 회사면 본인 메모리 디렉터리(`~/.claude/.../memory/project_*.md` 등)에서 해당 프로젝트 메모를 먼저 Read.

## 1. 병렬 리서치 (핵심 시간 절감 — 항상 병렬)
`deck-researcher` 서브에이전트를 **단일 메시지에서 facet별로 동시 호출**한다(가능하면 항상 병렬):
`company-product` · `jd-role` · `competitors` · `market-demand` · `benchmark-cases`
각 결과는 출처·신뢰도 태그(`[자사발표]`/`[제3자]`/`[추정]`/`[미검증]`)가 붙어 돌아온다. 캡처 추천 URL을 모은다.

## 2. 서사 설계 (이게 덱의 척추 — `slide-architecture.md` 참고)
- **페인포인트 → 전략 1:1 매핑**이 핵심 골격.
- 경쟁사 표에서 **공통 공백(빈틈)**을 찾고 → 그게 USP가 된다.
- 모든 전략에 **벤치마크 레퍼런스**를 붙인다(외부 사례 캡처).
- 마지막은 항상 **기대효과/KPI(단일 NSM)** + **왜 나(JD 자격 1:1 매핑)** + **지원동기/포부**.
- 사용자에게 슬라이드 순서(목차)를 먼저 확인받고 빌드 — 재작업이 가장 비싼 단계다.

## 3. 에셋 캡처 (Playwright MCP, 실측)
캡처 추천 URL을 Playwright MCP(`browser_navigate` + `browser_take_screenshot`)로 실측 캡처 → `<프로젝트>\images\` 또는 `assets\`에 저장.
- 브라우저 MCP가 자주 락 걸림 → 멈추면 chrome 프로세스 kill 후 재시도.
- 캡처는 단일 브라우저 인스턴스라 **순차** 처리(리서치와 달리 병렬 X).
- Playwright MCP가 없으면 이 단계는 건너뛰고 사용자가 수동 캡처를 첨부하게 한다.

## 4. 덱 빌드 (`deck-template.html` 복제 → 채우기)
- 단일 HTML, **1280×720, 키보드 네비(←→ Space, F 풀스크린), 진행바, 자동 스케일**. 템플릿의 CSS 토큰·컴포넌트(`.card .metric table .quote .flow .shot .foot` 등) 재사용.
- **톤은 회사 브랜드로 오버라이드**: `:root`의 `--blue/--violet` 등 액센트를 회사 브랜드 컬러로 교체. 폰트는 Pretendard 고정.
- **출처 규율**: 모든 외부 수치는 슬라이드 하단 `.foot .src`에 출처+신뢰도. 자사발표 vs 제3자 구분 표기. 단정 금지 항목은 단서 명시.
- 용어는 쉬운 우리말(전문용어는 괄호 병기). 헤드라인 강조는 슬라이드당 한 곳.
- 빌드 후 로컬 렌더 확인이 필요하면 로컬 정적 서버로 포트를 띄워 확인(`file://`는 일부 기능이 막힘).

## 5. 배포 (`deck-deployer` 서브에이전트 — 선택)
완성 폴더를 넘기면 **본인 GitHub Pages repo**에 복사·push·200 검증·URL 반환까지 처리. 배포가 필요 없으면 로컬에서 HTML을 열어 발표해도 된다. (deck-deployer는 본인 환경에 맞게 repo·git 경로를 설정해야 한다 — 에이전트 설명 참고.)

## 6. 회고 메모리 저장 (다음 세션 재사용)
본인 메모리 디렉터리에 `project_<회사>_<용도>_deck.md` 생성/갱신:
회사·직무·thesis, 슬라이드 구성, 핵심 수치+출처, 캡처 목록, 배포 URL+commit, 주의사항(공개금지 등). 인덱스 파일에 한 줄 포인터 추가. 내부데이터 포함 시 **public 배포 금지** 명시.

## 자주 쓰는 변주
- **컬쳐핏/2차**: 회사 사각지대 진단 + why-me 중심(전략 비중↓).
- **과제형**: 과제 문항 답안을 슬라이드로 + 별도 분석 덱.
- **BM 확장**: 구독/리테일미디어/수수료 등 매출모델 슬라이드 + 정량 프로젝션(가정·벤치마크 명시).
